## ğŸ¯ About

**Teachsly** is a comprehensive open-source e-learning platform designed for English language learning. It provides a complete solution for managing courses, students, teachers, exams, homework, media content, and lesson planning. The platform supports virtual classes through Zoom integration and includes payment processing capabilities.

4. **Pull Amplify backend** (if backend is already deployed)
   ```bash
   amplify pull
   ```

## âš™ï¸ Configuration

### Environment Variables

Create a `.env` file in the root directory with the following variables:

```env
# GraphQL API Endpoint (optional - defaults to production API)
REACT_APP_GRAPHQL_API_ENDPOINT=your-amplify-graphql-endpoint

# Environment (prod, qa, dev)
REACT_APP_ENV=dev

# Google Analytics Tracking ID
REACT_APP_GOOGLE_ANALYTICS_TRACKING_ID=your-tracking-id

# Zoom Integration
REACT_APP_ZOOM_API_ENDPOINT=your-zoom-api-endpoint
REACT_APP_ZOOM_CLASSES_LINK=https://your-zoom-classes-link.com
```

### AWS Amplify Configuration

The `src/aws-exports.js` file is automatically generated by AWS Amplify. It contains:

- AppSync GraphQL endpoint
- Cognito User Pool configuration
- S3 bucket configuration
- Region settings

**Note**: This file is auto-generated and should not be manually edited. It's updated when you run `amplify pull` or deploy changes.

## â˜ï¸ AWS Services

This project uses AWS Amplify to manage the following services:

### 1. **AWS AppSync** (GraphQL API)

- **Purpose**: Backend API for all data operations
- **Authentication**: Amazon Cognito User Pools (primary), API Key (additional)
- **Features**: Real-time subscriptions, CRUD operations, filtering, pagination

### 2. **Amazon Cognito**

- **User Pools**: User authentication and management
- **User Pool Groups**: Role-based access control (Admin, Teacher, Student)
- **Features**:
  - Email-based authentication
  - Password policy (minimum 8 characters)
  - Email verification
  - Custom message Lambda trigger

### 3. **Amazon S3**

- **Purpose**: File storage for media uploads
- **Features**:
  - Secure file uploads
  - Private access with signed URLs
  - Content type detection

### 4. **AWS CloudFront**

- **Purpose**: CDN for serving media files
- **Features**:
  - Signed URLs for secure access
  - Fast content delivery
  - Environment-based URL generation

### Backend Schema

The GraphQL API includes the following models:

- **Course**: Course management with schedules and Zoom integration
- **User**: User profiles with roles and groups
- **Exam**: Exams and homework with question pools
- **ExamAttempt**: Student exam submissions and results
- **Media**: Media files (videos, documents, links)
- **MediaFolder**: Folder organization for media
- **LessonPlan**: Lesson planning and organization
- **Payments**: Payment tracking (Mercado Pago)
- **Booklets**: Course materials/products
- **ClassAttendance**: Attendance tracking
- **PlacementResults**: Placement test results

## ğŸ‘¥ User Roles

The platform supports four user roles:

1. **Admin**

   - Full system access
   - Manage courses, users, exams
   - View all content and analytics

2. **Teacher**

   - Create and manage courses
   - Create exams and homework
   - Grade student submissions
   - Manage media content
   - View student progress

3. **Student**

   - View assigned courses
   - Take exams and homework
   - View media content
   - Track own progress

4. **Booklet Student**
   - Similar to Student role
   - Access to booklet-specific content
